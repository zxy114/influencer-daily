Karpathy 的 LLM 辅助编码分层工作流
工作流
实用性
15/15

Karpathy 分享了他日常使用 LLM 编码的分层策略——从 Tab 补全到 GPT-5 Pro 疑难调试，每一层都有明确的使用场景。

  • 基础层 (~75%) — Cursor tab 自动补全，自己写代码块比文字描述更高带宽
  • 第二层 — 选中代码段要求 AI 做具体修改
  • 第三层 — Claude Code / Codex 处理较大功能块（不用 YOLO 模式）
  • 最终层 — GPT-5 Pro 处理卡壳 10+ 分钟的隐蔽 bug

他还指出 AI 代码的常见问题：过度防御、抽象过度、缺乏"品味"。

💡 如何应用

不要依赖单一 AI 工具。建立自己的"编码分层"——日常补全用 Tab，具体修改用选中编辑，大块功能用 agent，疑难 bug 用最强模型。

AI Agent 架构设计：主 Agent vs 子 Agent vs 工具
工作流
实用性
12/15

Bryan Johnson 拆解了 AI Agent 系统设计的三种角色及各自适用场景：

  • 主 Agent — 保持工作流上下文和状态，但会遇到"上下文腐烂"
  • 子 Agent — 擅长特定知识、专用工具、工作流特定角色
  • 工具 — 需要可靠执行相同操作或与外部系统交互的场景

核心启发：想想人类团队如何完美完成任务，然后实现其核心要素。没有银弹，每个方案都有权衡。

💡 如何应用

设计多 Agent 系统时，用这个框架决定什么放主 Agent、什么拆子 Agent、什么做成确定性工具调用。

Karpathy：System Prompt Learning — LLM 学习的新范式
思维Prompt
实用性
12/15

Karpathy 提出 LLM 学习可能缺少一个关键范式——System Prompt Learning：

  • 预训练 → 知识
  • 微调 → 习惯性行为
  • 系统提示学习 → 问题解决策略

大部分人类学习更像改变"系统提示"——遇到问题，想明白，然后用明确的方式"记住"下次该怎么做。LLM 就像《记忆碎片》里的主角——我们还没给他们配备"记事本"。

💡 如何应用

构建 AI 应用时，考虑给 LLM 一个"策略笔记"——让 AI 把解决问题的经验写成系统提示，比重新训练高效得多。
