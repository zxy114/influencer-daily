「少规则、多目标」共识形成：OpenAI 官方 + Cognition 工程师同向推同一种 agent 心智

💡 如何应用 ① 「描述结果、不描述步骤」从 petergyang 一家之言升级为 OpenAI 官方背书的行业共识——petergyang 7/12 推的是「Cognition 团队（Devin 制造者）实战经验」、OpenAI 7/9 推的是「GPT-5.6 官方设计哲学」——两边独立得出同一个结论：写 prompt 时少给步骤，多给目标 + 关键约束；② 从 7/13 报告 #4 的「3 个方法论」升级为「3 步 prompt 重写法」：（a）找出你最近一个「步骤型 prompt」（例：「先搜 X、再分析 Y、然后写 Z」）→ （b）改写成「目标型 prompt」（例：「研究 X 主题，输出 1000 字报告，引用至少 5 个来源」）→ （c）保留硬约束，删掉软步骤（如「不能编造数据」「不能超 800 字」保留，「先查 A 再查 B」删掉）——OpenAI 自己就是这么设计 GPT-5.6 的；③ 3 个立即能套的场景：（a）写长文——以前"先列大纲、再写引言、再写正文"——新 prompt："针对 X 受众写一篇 2000 字深度文，论据来自 [附件]，3 个小标题，语气参考 [示例]"；（b）做调研——以前"查 A 网站、查 B 论文、查 C 报告"——新 prompt："研究 X 主题最新进展，输出一份对比表格，列 3 个最佳方案+各 3 个优劣点"；（c）做视频脚本——以前"先写 hook、再写 3 段、最后写 CTA"——新 prompt："写 5 分钟 YouTube 视频脚本，目标受众 X，开头 10 秒必须留住人，结尾有具体 CTA"——让 AI 自己决定结构；④ 为什么这是 2026 年最重要的 prompt 心智：步骤型 prompt 在 GPT-5.5 时代是必要的（模型推理弱），目标型 prompt 在 GPT-5.6 时代是更优的（模型推理强）——不升级 prompt 模式 = 浪费新模型的能力。
