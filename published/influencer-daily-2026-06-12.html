💡 工作流 & 思维

8非编码 Agent 也要多用 Bash 工具
工作流
"I've done dozens of calls with companies making general agents over the past few weeks and my advice generally boils down to: 'use the bash tool more.'"
来源：@trq212
实用性：⭐⭐⭐⭐⭐
如何应用：给 Agent 开放 bash 工具，让它执行 grep、sed、curl、jq 等命令处理文件和 API，大幅提升非编程类 Agent 的实用性。
9用 Hooks 让 Agent 过夜运行 6-8 小时
工作流
"Daisy's tip for getting Sonnet 4.5 to run for 6-8 hours overnight: use hooks. She adds hooks for linting & testing to make sure that Claude runs the tests and prompts it to not stop until they pass."
来源：@trq212
实用性：⭐⭐⭐⭐
如何应用：在 Claude Code 中配置 Lint/Test Hook，让 Agent 在代码出错时自动修复并重试，实现真正的无人值守过夜开发。
10Agent 应该使用文件系统表示状态
思维
"Your Agent should use a File System. This is a hill I will die on. The file system is an elegant way of representing state that your agent could read into context & allowing it to verify its work."
来源：@trq212
实用性：⭐⭐⭐⭐
如何应用：让 Agent 把中间结果写入文件而非全塞进上下文，既节省 token 又方便验证，适合长流程自动化任务。
11如何让 Agent 自主决策并派生子 Agent
思维
"An agent should be able to be given tools and decide what to do itself, including spinning off subagents, like Claude Code."
来源：@trq212
实用性：⭐⭐⭐⭐
如何应用：设计 Agent 时赋予工具自主选择权，让它根据任务复杂度决定是否启动子 Agent 并行处理，而非写死线性工作流。
12如何构建不是"壳"的 AI 产品
思维
"Some quick thoughts on how to build AI products that are not wrappers: 1. Study user workflows deeply: tools, data sources, hacks, manual activities. 2. Identify the challenges."
来源：@realmadhuguru
实用性：⭐⭐⭐⭐⭐
如何应用：深入调研用户的真实工作流，找出 AI 能切入的痛点，而不是简单地在现有产品上套一层 ChatGPT。
