Anthropic 内部"隔夜跑 Agent"工作流 — 睡前派活、醒来清空 backlog
🛠️ 工作流 🛠️ 教程

来源: @petergyang · 推文链接 → ⭐ 9/10
如何应用：内容创作者最大的杠杆不是"白天多打几个 prompt"，而是"睡前派活、睡醒收割"。把"批量调参、生成图片、跑长数据分析、爬取/整理多平台素材"这类可异步的任务丢给长时运行 Agent，本周就能搭一个最小版：每晚 23:00 派活，早 8:00 检收结果，省 2 小时/天。

Midjourney 隐藏指令 --preview + --sref random — 提前尝 V8.2 + 24 倍速风格海选
🧩 工具
图像世界两条快速更新：在 prompt 里加 --preview 可提前看 V8.2 的美学与个性化效果。同时把大批量草稿模式更新成支持 --sref random，这样你探索风格空间比以前快 24 倍。
来源: @midjourney · 推文链接 → ⭐ 8/10
如何应用：做封面、缩略图、风格探索时，加 --preview 少走 1-2 轮废稿；做"风格海选"时把 --sref random 塞进批模式，10 分钟内能拿到 200 张风格迥异的候选，比手动调 --sref 编号快一个数量级。立刻就能用。

Claude Tag — 在 Slack 里把 Claude 当"团队成员" @ 一下就派活
🧩 工具 🛠️ 工作流
推出 Claude Tag，团队跟 Claude 协作的新方式。在 Slack 里，Claude 作为"团队成员"加入，能访问你选定的频道和工具。@ 它一下就能委派任务，你可以专注别的工作。
来源: @AnthropicAI · 推文链接 → ⭐ 8/10
如何应用：创作者小型团队/社群运营的"协作 AI"终于来了。在现有 Slack workspace 里把 Claude tag 进来，给它配几个频道（#选题脑暴、#评论回复、#数据复盘），以后 @Claude + 一句话指令（"基于今早推文生成 5 个下周选题"）就能分流掉 30% 的脑力负荷。

Kimchi Coding 路由 + M3 — 不同复杂度任务自动派给不同模型
🧩 工具 🛠️ 工作流

来源: @MiniMax_AI · 推文链接 → ⭐ 6/10
如何应用："一人公司"自建 coding 工作流时，最痛的就是"小脚本用 GPT-5 Pro 太贵，大改需求用 GPT-3.5 答非所问"。Kimchi 的思路是"按任务复杂度路由" — 内容创作者不需要自建路由，可以直接用 Cast AI 的 Kimchi，或者用 LiteLLM/Router 自己在本地搭个 3 档路由（cheap/mid/premium），月度 API 费立省 40%。
