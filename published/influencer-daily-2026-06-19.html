Applied AI 企业级 playbook — 五大组件深度拆解

来源：@levie · 推文 思维 工作流 实用性：10/10

Box CEO Aaron Levie 深度总结了企业级 Applied AI 的五层 playbook：

  1. 构建桥接功能 — 在智能与工作流之间搭建专属界面，捕获 Agent 所需的上下文数据
  2. 模型路由器 — 平衡前沿模型与低成本模型，不同任务不同路由
  3. 落地实施支持 — FDE 帮助企业完成数据清洗、流程重构、评估和 SLA
  4. 领域专属 GTM — 每个行业有不同的话术、合规、安全要求和系统集成伙伴
  5. 领域专长护城河 — 越深入某一领域，越能讲客户听得懂的语言
如何应用：如果你在做企业级 AI 产品或服务，这五层框架可以直接作为产品规划清单。即使个人创作者，也能借鉴「模型路由器」思路——用最强模型做规划审查，用便宜模型跑批量任务。

5步搭建 AI 人生导师 — 用 Codex/Claude Code 做个人顾问

来源：@petergyang · 推文 教程 实用性：8/10

5步教程将 Codex 或 Claude Code 变成个人人生/职业导师，需要 4 个配置文件：

  • SKILL.md — 导师角色定义和行为规范
  • plan.md — 你的目标、原则、精力管理
  • learnings.md — 历史对话中提炼的洞察
  • eval.md — AI 给建议前必须跑的检查清单
如何应用：直接在 Claude Code 中创建 /advisor skill，把以上4个文件放进 skill 目录。每聊一次，learnings 会自动积累。适合做周复盘、职业决策、目标跟踪。

模型路由策略 — 为什么能路由到最佳 AI 模型的层会越来越值钱

来源：@levie · 推文 思维 实用性：6/10

三大原因支撑模型路由的价值增长：

  • 成本优化 — 同一任务中，规划用前沿模型，执行用便宜模型
  • 能力最大化 — 不同模型擅长不同领域（工具调用、编程、知识工作）
  • 风险对冲 — Fable 出口管制事件提醒：不要依赖单一模型提供商
如何应用：在你的 coding harness 中配置多模型策略：GPT 做规划 → Claude 做执行 → 开源模型做批量处理。Cursor/Claude Code 都支持这种模式。
